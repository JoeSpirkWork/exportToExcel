Getting Started

Bentley MicroStation SDK

Template Overview

Always start Visual studio from an administrator MicroStation CONNECT Edition SDK shell.
Simply type in devenv and press enter.

Build Your Project

To build the project successfully no changes are needed to the code. Simply press F7 or press Build>Build Solution. Be sure to look in the output window to verify that the build was successful.

Debug with MicroStation

To Debug in MicroStation, press F5 or click the Local Windows Debugger/Start button.

Load Your Application

  1. To load your application in MicroStation go to the Utilities>MDL Applications menu, select your application, and click load. You can also key-in mdl load PROJECTNAME.
  2. After you load the application, for C++ click the key-in button in the MDL Applications menu, or for C# you can key-in PROJECTNAME COMMAND to run the program.
  3. C++ applications can be unloaded from the MDL Applications menu, but C# applications cannot be unloaded so MicroStation must be restarted in order to edit your project.

Personalization

  1. Change the @description comment to a helpful description of your method.
  2. Replace Bentley with your name and the date you last modified that function.

Insert Items

If you want to add a new Bentley MicroStation SDK file, you can right click on the project in the Solution Explorer, select Add>New Item and choose the appropriate file type.

Microstation SDK EULA is located: \..\Program Files\Bentley\MicroStationCONNECTSDK\Eula.pdf

Best Practices

  • Insert the Bentley Header into new files
  • Use "//" in multi-line comments
  • Avoid long functions and deeply nested code
  • Do not use TABs, editors should use four spaces
  • Long lines should be split at 150 columns
  • Use abbreviations only where they are common and completely unambiguous
